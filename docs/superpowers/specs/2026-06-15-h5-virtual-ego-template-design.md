# H5 虚拟热点人士 ego · 模板设计

> 2026-06-15 · 设计阶段 · 待实现

## 1. 概述

### 1.1 目标

为 ego 产品设计一个可复用的 H5 宣传模板——"虚拟热点人士账号"。用户在社交媒体上看到热点人士的 ego 内容 → 进入 H5 旁观体验 → 被引导下载 ego app 建立自己的星图。

### 1.2 核心思路

不是广告，是体验。用户以第三人称旁观一个热点人士的 ego 空间——看 ta 的星图、星座、过去说过的话——感受到"原来内心可以长成这样"，从而想要建立自己的 ego。

### 1.3 设计原则

- **不登录**：H5 不做任何登录/注册，零摩擦进入
- **不硬推**：体验本身有完整价值，下载是自然结果
- **可复用**：一套模板，填入不同人物的数据即可生成新页面
- **视觉一致**：深空 + 星光 + 金色脉冲，与 ego app 视觉体系一致
- **移动端优先**：375px 基准设计

## 2. 用户路径

```
社交媒体内容（截图/录屏/卡片）
    │
    ▼
H5 旁观体验（单页，4 段式滚动）
    │
    ▼
App 下载 → 注册 → 建立自己的 ego
```

## 3. H5 模板结构

单页纵向滚动，4 个 Section。

### 3.1 S1 · 星图全景

**展示内容**：人物的完整星图，3 个 Constellation。

**视觉状态**：
| 状态 | 视觉 | 含义 |
|------|------|------|
| 已成型星座 | 亮星 + 实线连线 + 标签名 | AI 识别到多颗 Star 属于同一主题 |
| 正在成型 | 弱光脉冲 + 虚线 + "隐约有什么…" | 2 颗 Star 相关但不足以聚类 |

**交互**：点击任意星座 → 展开 Section 2。

**标题文案**：`如果 {人物名} 有 ego` / `ta 的星图长这样`

### 3.2 S2 · 星座展开

**展示内容**：1 个 Constellation 的详情。

**包含三块**：
1. **Constellation 名** — 如 "✦ 关于等待"
2. **✦ 我发现** — Constellation 级的 AI 观察卡片（金色边框）
3. **星星列表** — 3 颗 Star，每颗显示 topic + 日期 + 连续表达次数

**交互**：点击任意 Star → 展开 Section 3。顶部 "← 星图全景" 返回。

### 3.3 S3 · 星内 Trace

**展示内容**：1 颗 Star 包含的完整 Trace。

**Star 本质**：一个 Trace（一次完整的写作会话）被收进星图后的形态。包含 4-6 个 `<Moment, Echo, Insight>` 元组，用 "↓ 顺着再想想" 连接。反映用户在一段连续时间内层层递进的思考过程。

**每个元组包含**：
- **Moment**：用户写下的原话（斜体，左边框金色渐变）
- **Echo**：匹配到的历史相似话语（"你之前也说过类似的"）+ 日期
- **Insight**：AI 对该 Moment 的第二人称观察（"✦ 我发现"）

**额外展示**：
- Star 主题（AI 基于整个 Trace 提炼）
- 对话入口：`💬 和那段时间的 ta 说说话`（紫色卡片）

**交互**：顶部 "← {Constellation 名}" 返回。对话入口暂为展示（完整对话功能在 app 内）。

### 3.4 S4 · 转化 CTA

**展示内容**：用刚才展示的具体内容做镜子，引导用户建立自己的 ego。

**文案结构**：
```
你看完了他的星图。
他的 Moment，他的 Echo，他的 Constellation。

他用"等"写成了一颗星。
他让"等"变成了"内耗"，又变成了"别扭"，
最后变成了"我看见自己"。

你的"等"，会变成什么？

你写下的每一句 Moment，
都会在某一天变成自己的 Echo，
然后被收进一颗 Star，
和其他的 Star 一起，连成只属于你的 Constellation。
```

**双 CTA**：
- 主动按钮：`建立我的 ego →`（金色描边圆角按钮）
- 次要文字：`嗯，先看看别人的`（极轻灰字）

## 4. 数据模型映射

H5 模板中的 UI 元素与 ego 后端实体的对应关系：

| UI 元素 | 后端实体 | 关系 |
|---------|----------|------|
| S1 星图上的星座 | Constellation | 含 name, star_ids[], constellation_insight |
| S2 星座内的星星列表 | Star | 1 Constellation → N Stars |
| S3 Moment 卡片 | Moment | 1 Trace → N Moments |
| S3 Echo 卡片 | Echo | 1 Moment → 1 Echo（含 matched_moment_ids[]） |
| S3 ✦ 我发现 | Insight | 1 Moment → 1 Insight |
| S3 Star 主题 | Star.topic | AI 基于整个 Trace 提炼 |
| S2 ✦ 我发现 | Constellation.constellation_insight | AI 基于整个 Constellation 的 Stars 提炼 |
| S3 对话入口 | Star（非 Constellation） | ChatSession 绑定 star_id |

## 5. 数据规格

单个 H5 页面（一个人物）需要生产的数据量：

| 实体 | 数量 | 说明 |
|------|------|------|
| Constellation | **3** | S1 展示，2 已成型 + 1 正在成型 |
| Star / Constellation | **3** | S2 展示，每颗可点击 |
| **Star 总计** | **9** | 3 × 3 |
| Moment / Star（Trace） | **4-6** | S3 展示，连续表达序列 |
| **Moment 总计** | **36-54** | 9 × (4~6) |
| Echo | **36-54** | 每 Moment 1 条，匹配自更早的 Moment |
| Insight | **36-54** | 每 Moment 1 条 |
| Constellation Insight | **3** | S2 展示 |
| Topic Prompt | **9** | 每 Constellation 3 条 |

## 6. 技术方向

- H5 为独立前端页面，不耦合 ego Flutter 客户端
- 数据通过 JSON 配置驱动模板渲染
- 交互模式：展开/折叠 + 滚动，不跳页
- 视觉实现复用 ego 设计系统（深空背景、星光动效、金色主题色）
- 移动端优先，375px 基准，适配 750px（2x）
