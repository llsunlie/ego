# About EGO Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add "About EGO" section with git-tag-derived version to the setting page.

**Architecture:** `make version` generates `version.dart` from `git describe --tags`; setting page imports and displays the `appVersion` constant in a new "关于" section between account info and logout.

**Tech Stack:** Makefile, Dart/Flutter, git

---

### Task 1: Add `make version` target and generate version.dart

**Files:**
- Modify: `Makefile`
- Create: `client/lib/core/version.dart`

- [ ] **Step 1: Add `version` target to Makefile**

Append to `Makefile` after the `sqlc` target:

```makefile
.PHONY: version
version:
	@echo "// Generated by \`make version\`. Do not edit." > client/lib/core/version.dart
	@echo "const String appVersion = '$$(git describe --tags --always)';" >> client/lib/core/version.dart
	@echo "version generated"
```

- [ ] **Step 2: Run `make version` to generate version.dart**

Run: `make version`
Expected: creates `client/lib/core/version.dart` with the current git describe output.

- [ ] **Step 3: Verify generated file**

Run: `cat client/lib/core/version.dart`
Expected: Shows `const String appVersion = 'v0.1.9-...';` (or whatever git describe returns).
